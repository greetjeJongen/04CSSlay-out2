Styling a list for navigation

It makes perfect sense to use a ul for a main navigation block. After all, isn't that a list of links?

So we start with the html file “navigation.html”, which is essentially a ul with 5 li wrapped in a nav element. The only thing you are allowed to change in this file is adding a stylesheet link in the head.

Version 1: nav for small screen

Make a new CSS file (“style_mobilefirst.css”) to obtain screenshot1.

A few remarks:

  • The different li elements are seperated by a small line
  • The active link (the page where you are right now) has a ‘reversed effect’: dark letter on light background.
  • The complete box of the li has to be clickable.
  • Hovering over a link produces a color background color change. Make this even more nice by adding a transition of .3 seconds! (Documentation: W3Schools on transition)

Version 2: Horizontal nav with display: inline-block

Make the list items inline-block to obtain the result in “screenshot2”.

There might be a problem with your solution to version 2. Perhaps you can spot a small gap between the list items. Try to find out where this gap comes from by using the developer tools (of FirefoxDevEdition).

If you try to fill the entire width of the screen with the five list items by setting the width of every li to 20%, you see how annoying this little gap between the li's becomes. Five times 20% + 4 times a small gap is greater than 100%.

This problem is documented in an article on CSS-tricks, which also provides some solutions.

Try some of these solutions which involve changing the html code of the list somewhat. The result should look like “screenshot3”, with no gap between the list items.

Make the list items inline and not inline-block. Describe the differences. (Documentation: medium.com)

Ultimate Version: Horizontal nav with flex

The previous versions are OK, but they don't give you a lot of flexibility. Flexbox to the rescue! Might we suggest you use flexbox at least in one place in your own site? The main navigation would be an excellent choice.

Change your CSS code to obtain “screenshot4”. Make the ul a flex container. Play around with different values for justify-content.

Make the flex items a third the size of the browser window and get the result of “screenshot5” in only of few lines of added code.
