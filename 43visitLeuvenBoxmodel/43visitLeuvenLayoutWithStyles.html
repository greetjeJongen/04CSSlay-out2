Website “Visit Leuven”: Layout with Styles

So far you used CSS to style text, background etc. In this excercise we focus on blocks and their position in relation to each other. However, ‘real’ positioning using flex and grid is not yet discussed. We'll put those into practice next week.

The exercises below target CSS only. You have to extend the given CSS in order to obtain the required presentation/screenshots. Do not change any HTML, unless explicitly asked. In particular, adding extra id's or classes is prohibited.

In previous exercises, we asked you to limit the screen width up to 400px. This week, you should put the screen width to 600px (tablet portrait) and wider (last exercise). Make a copy of the CSS-for-small-devices since we will need it again in a few weeks.

Display: flex

Resize the browser window to 800px (small tablet landscape).

Documentation

  • https://css-tricks.com/snippets/css/a-guide-to-flexbox/
  • https://www.w3schools.com/css/css3_flexbox.asp

Exercise

Create a flexible horizontal navigation menu using display:flex. Take the following requirements into account:

  • The menu items should be evenly distributed over the total width of header
  • Not only the text of a menu-item should be clickable, also the (invisible) space around the text. In fact every click in the overlay zone should lead to a webpage.
  • When a menu item is hovered, the item is highlighted by a border.
  • Valid CSS.
  • Do you prefer a smooth transition or more subtle effects? Try to implement it yourself, take a look at https://www.w3schools.com/css/css3_transitions.asp.
